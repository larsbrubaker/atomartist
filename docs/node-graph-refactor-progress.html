The big change in one paragraph

The node-graph engine used to pin sockets to the NodeDef trait via &'static str keys, with edges identified by (NodeId, &'static str). That meant per-instance dynamic sockets, rename-without-edge-churn, and reorder were all blocked. We pivoted to instance-owned sockets with stable UIDs: every node now carries a Vec<Socket> for its inputs and outputs, each socket has a graph-unique SocketUid(u64), and noodles reference those uids exclusively. Names are display-only and can change freely without touching noodles. This unlocked the Blender-style dynamic-input Output node (one empty trailing slot, adopts source name+type on connect, appends a new empty slot, removes on disconnect, duplicate-source rejection) and the same shape now powers Combine.

Vocabulary — get the names right

Stored in ~/.claude/projects/…/memory/atomartist_vocabulary.md so future sessions stay consistent. The user is sensitive to inconsistency here.

Node
A box in the graph. Types: NodeId, NodeInstance, NodeDef, NodeRegistry.
Graph
The whole DAG (the document).
Socket
Input/output connector — the dot/pin on a node row. Carries SocketUid, name, display_label, socket_type, optional.
input / output
Direction is implied by which list a socket lives in: NodeInstance.inputs vs .outputs.
Noodle
The ribbon between two sockets. Types: Noodle, NoodleEndpoint. Never "edge", "wire", "link", or "connection" for this concept.
Property
An editable scalar on a node row. PropDef, NodeProperties, node.properties.

Mesh-side "edge" (triangle edges in geometry code) stays "edge". The rule applies only to the noodle concept.

What landed (Stages 1 & 2)

Stage 1 — engine refactor

  • New socket primitives: SocketUid(u64), Socket { uid, name, display_label, socket_type, optional }, SocketUidAlloc (in atomartist-lib/src/graph/socket.rs).
  • NodeInstance owns sockets: inputs: Vec<Socket>, outputs: Vec<Socket>, cached_outputs: HashMap<SocketUid, PortValue>. Properties keyed by Arc<str> for runtime-allocated names.
  • Noodles use uids: Noodle { from: NoodleEndpoint, to: NoodleEndpoint } where endpoint is (NodeId, SocketUid).
  • NodeDef trait shrank to factory + behavior: instantiate(alloc) -> InstanceTemplate, evaluate(ctx) -> NodeOutputs, plus optional hooks validate_input_connection, on_input_connected, on_input_disconnected.
  • Granular socket-mutation API on Graph: rename_socket, relabel_socket, retype_socket, append_input_socket, append_output_socket, remove_input_socket, remove_output_socket, reorder_input_sockets, noodles_touching. (Lives in graph/socket_mutations.rs to keep graph.rs under the 800-line cap.)
  • Serialization v2 (graph_json.rs): per-node socket lists, noodles by uid, next_socket_uid for allocator resume. v1 saves no longer load (sanctioned break).
  • ~25 existing nodes ported to instantiate() + uid-keyed EvalCtx.

Stage 1i — cleanup

  • Deleted nodes/io/graph_output_node.rs. The unified Output node plays both viewport-anchor and subgraph-output-declarator roles.
  • Rebuilt SubgraphNodeDef to scan templates for Output's mirror outputs (skipping the internal __display__) as the published subgraph ports.
  • Rebuilt CombineNode on the dynamic-input mechanism (second consumer that validates the design). Placeholder type is Geometry3d (constrained), not None (wildcard) — proves the design generalizes.

Stage 2 — unified Output node

atomartist-lib/src/nodes/output_node.rs — Blender-style dynamic multi-input. Trailing empty slot type None (wildcard, accepts any source). On connect: adopts source name + type, formats display label as "{NodeTitle} - {socket_name}", mints a mirror output socket, appends a fresh empty. validate_input_connection rejects duplicate-source. on_input_disconnected collapses the slot + removes the mirror. evaluate mirrors each named input to its named output, plus merges every Geometry3d input into a synthetic __display__ output the viewport reads.

Engine support: SocketType::None is now a wildcard target type (any source can land on it). Symmetric direction not allowed — outputs always carry a concrete type.

Bug fixed during the port

Symptom: screenshot showed an Extrude→Output noodle ending in the middle/right of the Output node, not at the left-side Extrude - Geometry input.
Root cause: agg-gui-node-editor's noodle paint resolved endpoints by name only. The Output node has both an input named Geometry (adopted slot) and an output named Geometry (mirror). layout_node_with_connections emits output rows first, so .find(|s| s.name == …) hit the output side.
Fix: extracted resolve_noodle_endpoints in agg-gui/node-editor/src/widget/mod.rs; from filters by SocketSide::Output, to by SocketSide::Input. Two regression tests in widget/tests.rs — one demonstrates the bug, the other guards stale-noodle paint.

Open threads / known gaps deferred

  • Source-side hooks. No on_output_connected, on_output_disconnected, or validate_output_connection yet. Symmetric to the input-side hooks. Add when a node needs to react to its consumers (e.g. JS-style onConnectionsChange(OUTPUT)).
  • Source-side wildcards. JS treats output type * as "match any input". atomartist only supports target-side SocketType::None wildcard (used by Output's trailing slot). Outputs always carry concrete types.
  • Drag-reorder UI. Engine support is done (Graph::reorder_input_sockets, edges unaffected since they're uid-keyed). Canvas-side row drag handler is not built.
  • Rename UI for mirror outputs. Currently subgraph publish names match source-socket names (e.g. an Output node fed from Extrude.Geometry publishes a port named Geometry). If users want friendlier names like result, we need a rename UI. Graph::rename_socket already exists.
  • connectByType / connectByTypeOutput. JS convenience helpers. Not in our API; flagged inline in node_connections.rs. Add when a UI flow demands them.
  • Multi-mesh viewport. Stage 2's Output node merges all geometry inputs into one mesh on __display__. If we ever need per-input materials/picking, extend last_mesh_output to Vec<Arc<MeshGL>>.
